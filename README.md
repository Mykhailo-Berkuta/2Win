## [cite_start]📝 Специфікація вимог до програмного забезпечення (SRS) [cite: 2]

### [cite_start]1. Вступ (Introduction) [cite: 3]
* [cite_start]**Опис:** 2Win — це вебсервіс, призначений для аналізу спортивних подій та надання користувачам точних прогнозів і рекомендацій щодо ставок[cite: 4].
* [cite_start]**Замовник:** Facebook[cite: 5].
* [cite_start]**Проблема:** Продукт вирішує проблему високого фінансового ризику, з яким стикаються гравці (особливо новачки), що роблять ставки «наосліп» або базуючись на неперевірених даних[cite: 6].
* [cite_start]**Цільова аудиторія:** Кінцевими користувачами є технічно грамотні люди віком 18-40 років, які прагнуть підвищити свої шанси на виграш за допомогою автоматизованої аналітики та статистики[cite: 7].

### [cite_start]2. Загальний опис (Overall Description) [cite: 8]

#### [cite_start]Рівні користувачів[cite: 9]:
* [cite_start]**Гість:** може переглядати лише обмежену загальну інформацію та маркетинговий контент[cite: 10].
* [cite_start]**Зареєстрований користувач:** має повний доступ до прогнозів, детальної статистики, історії власних ставок та персональних налаштувань[cite: 11].
* [cite_start]**Адміністратор:** керує контентом, моніторить точність прогнозів та керує обліковими записами користувачів[cite: 12].

#### [cite_start]Зовнішні залежності[cite: 13]:
* [cite_start]**АРІ спортивної статистики:** джерела даних про матчі, склади команд та результати[cite: 14].
* [cite_start]**АРІ букмекерських платформ:** для отримання актуальних коефіцієнтів[cite: 15].
* [cite_start]**Платіжні системи:** для обробки підписок на преміум-функції[cite: 16].

### [cite_start]3. Функціональні вимоги (Functional Requirements) [cite: 17]
* [cite_start]**FR-01 [Must]:** Система повинна забезпечувати реєстрацію та автентифікацію користувачів (email/пароль)[cite: 18, 19].
* [cite_start]**FR-02 [Must]:** Система повинна автоматично генерувати прогнози на спортивні події на основі аналітичних алгоритмів[cite: 20].
* [cite_start]**FR-03 [Must]:** Користувач повинен мати можливість переглядати детальну статистику матчів і рейтинг команд[cite: 21].
* [cite_start]**FR-04 [Must]:** Система повинна відображати персоналізовані рекомендації для ставок на основі обраних видів спорту[cite: 22].
* [cite_start]**FR-05 [Should]:** Система повинна зберігати та відображати історію ставок та персональні налаштування користувача[cite: 23].
* [cite_start]**FR-06 [Should]:** Система повинна надсилати сповіщення (web-push або email) про нові важливі матчі[cite: 24].
* [cite_start]**FR-07 [Should]:** Користувач повинен мати можливість зберігати матчі в категорію «Улюблені»[cite: 25].
* [cite_start]**FR-08 [Could]:** Інтеграція з мобільними месенджерами (наприклад, Telegram-бот) для отримання миттєвих прогнозів[cite: 26].

### [cite_start]4. Нефункціональні вимоги (Non-Functional Requirements) [cite: 27]
* **Продуктивність (NFR-P):** Час відповіді системи не повинен перевищувати 1-3 секунди для типових операцій. [cite_start]Підтримка кількох тисяч одночасних сесій[cite: 28].
* **Безпека (NFR-S):** Обов'язкове шифрування даних (HTTPS/SSL). [cite_start]Відповідність стандартам GDPR щодо захисту персональних даних[cite: 29].
* [cite_start]**Зручність (NFR-U):** Адаптивний дизайн за принципом mobile-first (коректне відображення на смартфонах і десктопах у браузерах Chrome, Safari, Edge, Firefox)[cite: 30].
* [cite_start]**Надійність (NFR-R):** Рівень доступності (Uptime) системи >= 99%, особливо в періоди проведення великих турнірів та чемпіонатів[cite: 31].

### [cite_start]5. Обмеження (Constraints) [cite: 32]
* [cite_start]**C-01:** Платформа повинна бути реалізована як вебсервіс (Python/Node.js)[cite: 33].
* [cite_start]**C-02:** Обмеження за віком: доступ до сервісу дозволено лише особам 18+ років[cite: 34].
* [cite_start]**C-03:** Залежність від стабільності та лімітів сторонніх АРІ спортивної статистики[cite: 35].
* [cite_start]**C-04:** Дедлайн для випуску МѴР-версії становить кілька місяців[cite: 36].
* [cite_start]**C-05:** Юридична відповідність законодавству про азартні ігри в цільових регіонах[cite: 37].

### [cite_start]6. Глосарій (Glossary) [cite: 38]
* [cite_start]**Прогноз (Prediction):** розрахований алгоритмом імовірний результат спортивної події[cite: 39].
* [cite_start]**MVP (Minimum Viable Product):** мінімально життєздатний продукт з базовою функцією генерації прогнозів[cite: 39].
* [cite_start]**GDPR:** загальний регламент про захист даних, що діє в ЄС[cite: 40].
* [cite_start]**Коефіцієнт (Odds):** числовий показник, що визначає розмір можливого виграшу в букмекерській конторі[cite: 41].
