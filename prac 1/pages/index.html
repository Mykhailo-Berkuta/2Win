Вступ
Introduction
  • Про продукт: 2Win - це вебсервіс, призначений для аналізу спортивних подій та надання користувачам точних прогнозів і рекомендацій щодо ставок.
  • Замовник: Група 5-3
  • Проблема: Продукт вирішує проблему високого фінансового ризику, з яким стикаються гравці (особливо новачки), що роблять ставки «наосліп» або базуючись на неперевірених даних.
  • Цільова аудиторія: Технічно грамотні люди віком 18–40 років, які прагнуть підвищити шанси на виграш за допомогою автоматизованої аналітики та статистики.
2
Загальний опис
Overall Description
Рівні користувачів
  • Гість: може переглядати лише обмежену загальну інформацію та маркетинговий контент.
  • Зареєстрований користувач: має повний доступ до прогнозів, детальної статистики, історії власних ставок та персональних налаштувань.
  • Адміністратор: керує контентом, моніторить точність прогнозів та керує обліковими записами користувачів.
Зовнішні залежності
  • API спортивної статистики: джерела даних про матчі, склади команд та результати.
  • API букмекерських платформ: для отримання актуальних коефіцієнтів.
  • Платіжні системи: для обробки підписок на преміум-функції.
3
Функціональні вимоги
Functional Requirements
  • FR-01 Must Система повинна забезпечувати реєстрацію та автентифікацію користувачів (email/пароль).
  • FR-02 Must Система повинна автоматично генерувати прогнози на спортивні події на основі аналітичних алгоритмів.
  • FR-03 Must Користувач повинен мати можливість переглядати детальну статистику матчів і рейтинг команд.
  • FR-04 Must Система повинна відображати персоналізовані рекомендації для ставок на основі обраних видів спорту.
  • FR-05 Should Система повинна зберігати та відображати історію ставок та персональні налаштування користувача.
  • FR-06 Should Система повинна надсилати сповіщення (web-push або email) про нові важливі матчі.
  • FR-07 Should Користувач повинен мати можливість зберігати матчі в категорію «Улюблені».
  • FR-08 Could Інтеграція з мобільними месенджерами (Telegram-бот) для отримання миттєвих прогнозів.
4
Нефункціональні вимоги
Non-Functional Requirements
  • NFR-P  Час відповіді системи не повинен перевищувати 1–3 секунди. Підтримка кількох тисяч одночасних сесій.
  • NFR-S  Обов'язкове шифрування (HTTPS/SSL). Відповідність стандартам GDPR.
  • NFR-U  Адаптивний дизайн mobile-first. Підтримка Chrome, Safari, Edge, Firefox.
  • NFR-R  Uptime системи ≥ 99%, особливо в періоди великих турнірів.
5
Обмеження
Constraints
  • С-01  Платформа повинна бути реалізована як вебсервіс (Python/Node.js).
  • С-02  Обмеження за віком: доступ дозволено лише особам 18+ років.
  • С-03  Залежність від стабільності та лімітів сторонніх API спортивної статистики.
  • С-04  Дедлайн для випуску MVP-версії становить кілька місяців.
  • С-05  Юридична відповідність законодавству про азартні ігри в цільових регіонах.
